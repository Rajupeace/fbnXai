OPTION 2: MONGODB ATLAS (FOR ONLINE/REMOTE ACCESS)
====================================================
Setup:
1. Create account at: https://www.mongodb.com/cloud/atlas
2. Create a new cluster
3. Set up database user with strong password
4. Whitelist IP addresses in Network Access
5. Get connection string from Connect option

Connection String Format:
mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/fbn_xai_system

Configuration in backend/.env:
MONGO_URI=mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/fbn_xai_system?retryWrites=true&w=majority
MONGO_CONNECT_ATTEMPTS=5
MONGO_RETRY_DELAY_MS=2000

Requirements:
- MongoDB Atlas account (free tier available)
- Strong password (16+ characters)
- IP whitelisting configured
- Stable internet connection

Benefits:

- Scales automatically

SECTION 11: TROUBLESHOOTING GUIDE
================================================================================

ISSUE 1: MongoDB Not Connecting
================================
Error Message: "ECONNREFUSED" or "MongoDB Connection Error"

Solution:
1. Verify MongoDB is installed:
   Get-Service | Where-Object {$_.Name -like "*MongoDB*"}

2. Start MongoDB service:
   Start-Service MongoDB

3. Verify connection:
   mongosh

4. Check firewall:
   - Allow MongoDB through Windows Firewall
   - Ensure port 27017 is accessible

5. Run diagnostic:
   node test-db-connection.js


ISSUE 2: Port 5000 Already in Use
==================================
Error Message: "EADDRINUSE" or "Port already in use"

Solution:
1. Kill process on port 5000:
   Get-NetTCPConnection -LocalPort 5000 -ErrorAction SilentlyContinue |
   ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }

2. Or change port in backend/.env:
   PORT=5001 (use different port)

3. Restart backend server


ISSUE 3: Port 3000 Already in Use
==================================
Error Message: "EADDRINUSE" for port 3000

Solution:
1. Kill process on port 3000:
   Get-NetTCPConnection -LocalPort 3000 -ErrorAction SilentlyContinue |
   ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }

2. Or use different port (rare)


ISSUE 4: MongoDB Atlas Connection Fails
========================================
Error Message: "ENOTFOUND" or authentication error

Solution:
1. Verify connection string format:
   mongodb+srv://username:password@cluster.mongodb.net/database

2. Check credentials:
   - Username exists in Database Users
   - Password is correct
   - Special characters are URL-encoded

3. Whitelist IP:
   - Go to MongoDB Atlas → Network Access
   - Add current IP or 0.0.0.0/0

4. Check firewall:
   - Allow outbound connection on port 27017

5. Verify network:
   - Test internet connection
   - No VPN blocking required


ISSUE 5: Dependencies Not Installed
====================================
Error Message: "Cannot find module" errors

Solution:
1. Reinstall backend dependencies:
   cd backend
   rm -r node_modules
   npm install --legacy-peer-deps

2. Reinstall frontend dependencies:
   cd ..
   rm -r node_modules
   npm install --legacy-peer-deps


ISSUE 6: Frontend Compilation Errors
=====================================
Error Message: Babel parser errors or syntax errors

Solution:
1. Clear cache:
   rm -r node_modules/.cache

2. Restart React dev server:
   npm start

3. Check for merge conflicts in files

4. Verify all imports are correct


ISSUE 7: File Upload Not Working
=================================
Error Message: Upload fails silently or error returned

Solution:
1. Check multer configuration in backend
2. Verify upload directory exists:
   data/uploads/

3. Check file size limits (default 50MB)

4. Verify file permissions:
   chmod -R 755 backend/data/


ISSUE 8: Real-time Updates Not Working
========================================
Error Message: Data doesn't update in real-time

Solution:
1. Verify SSE client is connected:
   Check browser console for SSE connection

2. Check backend SSE endpoint:
   GET http://localhost:5000/api/sse

3. Verify firewall allows streaming

4. Check for proxy issues:
   - Disable reverse proxies temporarily
   - Check CORS settings


ISSUE 9: Login Not Working
===========================
Error Message: Authentication error or "Invalid credentials"

Solution:
1. Verify user exists in database:
   mongosh
   use fbn_xai_system
   db.Admin.findOne({username: "BobbyFNB@09="})

2. Check JWT secret in backend/.env:
   JWT_SECRET must be set

3. Clear browser cookies/cache:
   Ctrl+Shift+Delete in browser

4. Verify password is exact (case-sensitive)


ISSUE 10: Performance Issues
=============================
Error Message: Slow database queries or UI lag

Solution:
1. Check database indexes:
   mongosh → use fbn_xai_system → db.getIndexes()

2. Monitor database performance:
   MongoDB Compass → Performance tab

3. Optimize queries:
   - Add missing indexes
   - Use projection to limit fields

4. Check browser resources:
   F12 → Performance tab

5. Clear cache and restart:
   npm start (with cache clear)

SECTION 15: SECURITY BEST PRACTICES
================================================================================

AUTHENTICATION & PASSWORDS
==========================
✅ Current: Local development mode
❌ Change before production:

1. Update admin credentials:
   - Change default username and password
   - Use strong, unique credentials
   - At least 12 characters with mix of:
     * Uppercase letters
     * Lowercase letters
     * Numbers
     * Special characters

2. JWT Secret Management:
   - Generate strong random secret
   - Store securely in environment variables
   - Change periodically

3. Password Storage:
   - Passwords should be hashed (bcrypt)
   - Never store plain text passwords
   - Use salting for additional security


DATABASE SECURITY
=================

1. For MongoDB Atlas (Online):
   - Use strong database user password
   - Enable IP whitelisting
   - Use VPN for sensitive data
   - Enable encryption at rest
   - Enable backup encryption
   - Rotate credentials quarterly

2. For Local MongoDB:
   - Enable authentication if exposed
   - Use firewall to restrict access
   - Regular backup strategy
   - Monitor access logs

3. Data Protection:
   - Never expose sensitive data in logs
   - Sanitize user inputs
   - Use parameterized queries
   - Validate all input


API SECURITY
============

1. HTTPS:
   - Use HTTPS in production (not HTTP)
   - Install SSL certificate
   - Configure TLS 1.2+

2. CORS (Cross-Origin Resource Sharing):
   - Configure allowed origins
   - Restrict methods (GET, POST, etc)
   - Review in production

3. Rate Limiting:
   - Limit requests per IP
   - Protect against DDoS
   - Implement exponential backoff

4. Input Validation:
   - Validate all user inputs
   - Check data types
   - Sanitize strings
   - Prevent injection attacks


FILE UPLOAD SECURITY
====================

1. File Type Validation:
   - Only allow specific file types
   - Check MIME types
   - Verify file extensions
   - Scan for malware

2. File Size Limits:
   - Set maximum file size
   - Current limit: 50MB
   - Reject oversized files

3. Storage Security:
   - Store files outside web root
   - Use unique filenames
   - Implement access controls
   - Regular backups

4. Virus Scanning:
   - Consider antivirus integration
   - Scan uploaded files
   - Quarantine suspicious files


SESSION MANAGEMENT
==================

1. Token Expiration:
   - Set JWT expiration time
   - Force re-authentication
   - Implement refresh tokens

2. Session Timeout:
   - Auto-logout after inactivity
   - Clear session data on logout
   - Invalidate tokens on exit

3. Secure Cookies:
   - Use HttpOnly flag
   - Set Secure flag for HTTPS
   - Use SameSite attribute


COMPLIANCE & PRIVACY

   - GDPR compliance if EU users
